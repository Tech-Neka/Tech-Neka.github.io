ROXY BDC: SQL SCHEME


PURPOSE

The main purpose of the Tables Scheme is to specify the appropriate data types to use for the data research and to specify integer ranges, in view of setting constraints to prevent mistakes and outliers. But it is also a means for identifying, and a source of obtaining useful and invaluable information about the company and its Key Performance Indicators (KPIs). It is used furthermore, for identifying professional working terms which differ from industry to industry, company to company and analyst to analyst.


CUSTOMERS

ITEM DATA TYPE  
CUSTOMER_ID INTEGER     This is a unique identity number given to Roxy's Customers.
GENDER NVARCHAR(50) This is the gender description
ADRESS NVARCHAR(50) This is the office or home address of the client, but the best contact
PASSPORT_NO. NVARCHAR(50) This is the required identity card for buying FOREX globally
NATIONALITY NVARCHAR(50) This is the ISO-2 Code representing the clients Nationality
CUSTOMER_NAME NVARCHAR(50) These are the names of the customers
OCCUPATION NVARCHAR(50) This is the job position of the customer
CATEGORY NVARCHAR(50) This is the category the customer is classified in purchasing reasons
AGE_RANK INTEGER This is the age group to which the customer belongs
ACQUAINTANCE NVARCHAR(50) This describes how the client got to know about the firm.

FOREX PURCHASES

ITEM DATA TYPE
BUY_DATE NVARCHAR(50) This is the date, usually on a Monday, when vendors pay the Central Bank, in local currency, for the purchase of dollars
TXN_ID INTEGER This is a unique identity number for each purchase or sale transaction done by Roxy BDC.
BUY_DAY NVARCHAR(50) This is the day, usually a Monday, when the Naira is paid to the Central Bank for the purchase of dollars
FX_RATE DECIMAL(18,2) This is the rate of exchange of Naira to the dollar on the given date.
COLLECTION_DATE DATE This is the date, typically on a Wednesday, when Central bank disburses the purchased sum of $50,000 to each of the vendors.
COLLECTION_DAY NVARCHAR(50) This is the day, typically a Wednesday, when the Central bank disburses the purchased sum of $50,000 to each of the BDCs.
VENDOR_ID INTEGER This is an identity number given to represent the Forex vendor.
AMOUNT INTEGER This is the cash amount in dollars that was bought from the CBN or any other vendor

FOREX SALES

ITEM DATA TYPE
SALE_DATE DATE This is the date on which a Roxy licensed staff sells FOREX to a customer
TXN_ID INTEGER This is a unique identity number for each purchase or sale transaction done by Roxy BDC.
VENDOR_ID INTEGER This is an identity number given to represent the Forex vendor.
WEEK_NO INTEGER This is number of the week in which the forex is sold, using the current year's calendar.
WEEK_DAY NVARCHAR(50) This is the day of the week in which the forex is sold.
P_HOLS NVARCHAR(50) This is the column that lists the public holidays available in the current year.
CUSTOMER_ID INTEGER This is the ID of the customer buying the FOREX.
PURPOSE_ID INTEGER This is an identity number representing the reason for which the customer is buying the Foreign Exchange.   
CATEGORY_ID INTEGER This is an ID classifying the types of buyers based on their buying purpose
AMOUNT INTEGER This is the cash amount in dollars that was bought from the CBN or any other vendor
BUY_RATE DECIMAL(18,2) This is the rate at which Roxy bought the FOREX that's being sold
SALES_RATE DECIMAL(18,2) This is the sale rate determined by Roxy staff based on the buy rate and customers' buying power
BEST_RATE DECIMAL(18,2) This is the highest possible rate at which forex can be sold, which is 3.5% of the buy rate
USD INTEGER This is the code for the forex when it is for all other purposes
BTA INTEGER This is the Business Travel Allowance for customers travelling for business purposes and cannot exceed $5000
PTA INTEGER This is the Personal Travel Allowance for customers travelling for personal purposes and cannot exceed $4000

VENDORS

ITEM DATA TYPE  
VENDOR_ID INTEGER Number assigned to the vendor for numbering purposes only
VENDOR_NAME NVARCHAR(50) The full name by which the vendor is registered to trade
VENDOR_CODE NVARCHAR(50) A shortened form of the full business trading name. An abbreviation or alias.

CUSTOMER PURPOSES

ITEM DATA TYPE  
ID INTEGER This is a numerical value classifying the customer's purpose for the purchase
CATEGORY NVARCHAR(50) This is the main reason the customer desires the foreign Exchange
PURPOSE NVARCHAR(50) This is the specific reason for which the customer makes the transaction

CATEGORY

ITEM DATA TYPE  
CATEGORY_ID INTEGER This is the ID classifying the buyers based on their buying purposes. It is a primary key.
CATEGORY NVARCHAR(50) This is the category which the buyers are grouped into

AGE GROUP

ITEM DATA TYPE  
AGE_GROUP NVARCHAR(50) This is the age group to which each customer belongs
RANK INTEGER This is rank assigned to the age group for simplification of calculations

STAFF

ITEM DATA TYPE  
STAFF_ID NVARCHAR(50) This is the ID assigned to each staff member
NAME NVARCHAR(50) These are the staff names
GENDER NVARCHAR(50) This is the gender of each staff member.
FIELD NVARCHAR(50) This is the area of study or degree obtained by the given staff
DEPT NVARCHAR(50) This is the department that the staff member works in.
HOBBIES NVARCHAR(50) These are the hobbies each staff enjoys or the activities that motivate them
ADRESS NVARCHAR(50) This is the home adress of the staff
SALARY INTEGER This is the staff salary.

FULL CALENDAR

ITEM DATA TYPE
YEAR NVARCHAR(50) This is the current year of the transactions
MONTH NVARCHAR(50) This is the current month of the transactions
DATE INTEGER This is the current date of the transactions
DAY NVARCHAR(50) This is the current day of the week of the transactions
FULL_DATE DATE This is the full date, including the date, month and year of the transaction
WEEK_NO INTEGER This is the number or count of the current week of the transaction.
P_HOLS_CODE NVARCHAR(50) This is an abbreviation for the public holiday.
P_HOLS_NAME NVARCHAR(50) These are the full names of the public holidays given.
WEEK_NAME NVARCHAR(50) These are names given to the weeks based on their position or fetivities .
SEASON NVARCHAR(50) This is the current weather-season in which the transaction is taking place
