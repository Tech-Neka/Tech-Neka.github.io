BUILDING THE DATABASE

SAMPLING AND DATA COLLECTION

The data collected for this project is secondary data and was obtained from reliable sources including government websites and Kaggle.com. It was extracted using Excel's 'GET and TRANSFORM' data tools. It consists of mostly numerical data which are often ranked variables for easy computations, but also features nominal, ordinal and boolean data classes comprising decimal, date, and variable-character datatypes.

CLEANING THE DATA

The data was cleaned using Microsoft Excel's wonderful Data editing tools and features including: 'FILTER', 'FIND $ REPLACE', 'DELETE' and 'SORT' to remove bogus and ambiguous data, erase duplicate records, delete or correct other inconsistencies such as naming conventions or typographical errors; and lastly to filter off records with missing values.

SAMPLING

Sampling is the process of using statistical estimates computated from small subsets of a given population to make inferences about that population. These selected subsets ought to properly represent the population as much as possible and not be biased.

Sampling is classified into random and non-random sampling types. Random sampling is sub-divided into Simple, Systematic, Stratified and Cluster sampling while non-random sampling includes convenience, volunteer, judgement, snowball/networking, and quota types.

This project uses Judgement sampling based on a criteria that permits only countries that participated in all the Happiness Index surveys carried out by the Gallup Poll team within the specified timeframe from 2013 to 2019. It therefore consists a large sample size of about 90% of the entire population which means a very high degree of certainty for the survey results.

BUILDING THE SQL DATABASE

Next, the SQL database was built using the cleaned data. This was done by creating useful tables, views and indexes using SQL queries which were based on the key performance indicators (KPI's)to meet the research objectives.

The SQL software used for the project is the Microsoft SQL Server. The queries used to build the database include both simple and complex queries and utilise a wide variety of SQL tools and clauses such as 'SELECT', 'FROM', 'WITH', 'UNION', 'TOP', 'AVG', 'GROUP BY' and 'ORDER BY' among others.
